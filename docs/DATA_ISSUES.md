## 6. Что уже проверено и работает нормально

- **Driver ID:** разное написание «Jr.» / «Jr» и «Suarez» / «Suárez» после `NormalizeKey` даёт один и тот же идентификатор, поэтому дубликатов пилотов в БД из-за этого не создаётся.
- **Соответствие расписания и файлов событий:** для проверенных серий (NASCAR Cup, F1) ID в `schedules/*.json` соответствуют имеющимся файлам в `data/events/`.
- **API data-diff:** в проекте есть `GET /api/admin/data-diff` для сравнения количества результатов в JSON и в БД по сток-кар сериям — полезно для контроля консистентности после импорта.
